# Android App — Agent Instructions

## Tech Stack

- **Language:** Kotlin (target JDK 8 bytecode)
- **Min SDK:** 33 (Android 13+)
- **Build:** Gradle with Kotlin DSL (`build.gradle.kts`)
- **BLE:** Android native BluetoothGatt API
- **OTA:** Nordic DFU library (`no.nordicsemi.android:dfu`)
- **UI:** Material Components, ConstraintLayout, RecyclerView
- **No Jetpack Compose** — this app uses XML layouts and traditional Activities.

## Architecture

- **BleService** (foreground service): Owns BLE scanning, GATT connections, and coordinates all background work.
- **GattConnectionManager**: Handles low-level BLE operations (scan, connect, read/write/notify).
- **VapeStateTracker**: Parses coil messages, deduplicates events, posts to server.
- **ServerClient**: HTTP client with offline queue and retry logic.
- **DeviceRepository**: SharedPreferences-backed persistent device list.
- **StatusNotifier**: Foreground notification and local broadcasts.

IPC between service and activities uses local broadcasts and a bound service pattern.

## Coding Standards

- **Kotlin idioms:** Use `val` over `var`, data classes, `when` expressions, scope functions (`let`, `apply`, `also`) where they improve readability.
- **Null safety:** Leverage Kotlin's type system. Avoid `!!` — use `?.`, `?:`, or early returns.
- **Threading:** BLE callbacks run on the binder thread. Use `Handler(Looper.getMainLooper())` for UI updates. Use coroutines or executors for network/IO, not raw threads.
- **Permissions:** Always check and request BLE/notification permissions before use. Handle denial gracefully.
- **Resource cleanup:** Unregister receivers, close GATT connections, and cancel pending operations in lifecycle callbacks.

## BLE Protocol

The firmware advertises service UUID `189a9192-f68f-4ac4-962e-d70e7c3755a0`. Messages are plain ASCII strings on the coil characteristic:

- `COIL_A:STARTED` / `COIL_A:STOPPED`
- `COIL_B:STARTED` / `COIL_B:STOPPED`
- `NOT_RIPPED`

The name characteristic (`...a993`) allows reading/writing the device's user-friendly name.

## Server Communication

- Endpoint: `POST /vape-update` with `Authorization: Bearer <token>` header.
- Server URL and auth token are stored in SharedPreferences.
- Events are queued and retried if the server is unreachable (max 200 pending).

## Best Practices

- Keep Activities thin — business logic belongs in the service or helper classes.
- Don't leak Context references. Use `applicationContext` for long-lived objects.
- Log BLE errors with enough detail to debug (operation, status code, device address).
- Handle BLE edge cases: connection timeout, bond loss, GATT 133 errors, MTU negotiation failures.
- Test OTA flows on real hardware — the DFU library has strict sequencing requirements.
