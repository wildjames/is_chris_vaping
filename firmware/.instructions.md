# Firmware — Agent Instructions

## Tech Stack

- **Platform:** Seeed XIAO nRF52840 (Nordic nRF52840 SoC)
- **Framework:** Arduino (Adafruit nRF52 BSP via PlatformIO)
- **Language:** C++ (Arduino-flavoured)
- **BLE Stack:** Adafruit Bluefruit (wraps Nordic SoftDevice S140)
- **Build System:** PlatformIO
- **OTA:** Adafruit nRF52 DFU (via `adafruit-nrfutil`)

## Module Structure

| File | Responsibility |
|------|---------------|
| `main.cpp` | Setup, main loop, shared state |
| `bluetooth.cpp/h` | BLE service/characteristics, advertising, message sending, name persistence |
| `coils.cpp/h` | GPIO pin monitoring, coil state detection, debouncing |
| `sleep.cpp/h` | Power management (light sleep via SoftDevice, System OFF on long idle) |
| `ota.cpp/h` | OTA/DFU support utilities |
| `version.h` | Firmware version string (injected at build time) |

## Coding Standards

- **Keep it simple.** This is a small embedded project. Flat function-based architecture, not OOP hierarchies.
- **No dynamic allocation** after `setup()`. Pre-allocate buffers. Use fixed-size arrays.
- **Const correctness.** Use `const` for string literals, pin definitions, and configuration values.
- **Global state is OK** for this project — but keep it minimal and declare `extern` in headers.
- **Naming:** `camelCase` for functions and local variables, `UPPER_SNAKE_CASE` for constants and macros, `PascalCase` for types.
- **Comments:** Explain *why*, not *what*. The code should be self-documenting for the *what*.

## BLE Protocol

- Service UUID: `189a9192-f68f-4ac4-962e-d70e7c3755a0`
- Coil characteristic UUID: `5cf4a205-84e1-42ad-ac23-e5adc776a992` (notify)
- Name characteristic UUID: `5cf4a205-84e1-42ad-ac23-e5adc776a993` (read/write)

Messages are plain ASCII, null-terminated:
- `COIL_A:STARTED`, `COIL_A:STOPPED`
- `COIL_B:STARTED`, `COIL_B:STOPPED`
- `NOT_RIPPED`

## Power Management

- **Light sleep** (idle >60s): FreeRTOS task blocks on semaphore; SoftDevice keeps BLE alive; GPIO interrupt wakes.
- **System OFF** (idle >30min): Full chip shutdown; GPIO wake triggers a reset.
- Always reset `lastActivityTime` on coil activity or BLE connection events.

## Build & Deploy

```bash
# Build
make build VERSION=1.2.3

# Upload via USB
make upload VERSION=1.2.3

# Create DFU package for OTA
make dfu-package VERSION=1.2.3
```

Without `VERSION=`, the firmware reports `<dev>`.

## Best Practices

- Test on real hardware. The BLE stack and power management don't work in simulation.
- Keep BLE message payloads short — MTU is limited and notifications are single-packet.
- Debounce GPIO inputs to avoid spurious coil events.
- Avoid blocking `loop()` for more than a few ms — the SoftDevice needs CPU time for BLE housekeeping.
- Use `Serial.printf()` for debug output, but wrap verbose logging in `#ifdef DEBUG` so it doesn't bloat release builds.
- Flash storage (for device name) uses Adafruit's InternalFileSystem — always check for write failures.
